Team Workflow

The GitHub Push & Pull Workflow

Follow the wire top to bottom — from creating your office account to deploying on Vercel.

📦 github.com/payprice-design/Push-and-Pull-Knowledge-Base
The everyday loop
🌿 Branch ✏️ Commit ⬆️ Push 🔀 PR ✅ Merge 🔄 Sync ↺
1
Setup · once

Create your GitHub office account

Use your company email so you can join the payprice-design organization.

  1. Go to github.com/signup and sign up with your.name@tiket.com (not a personal email).
  2. Pick a recognizable username, e.g. firstname-tiket.
  3. Verify the email GitHub sends to your office inbox.
  4. Enable Two-Factor Authentication under Settings → Password and authentication.
  5. Send your username to your org admin to get invited to payprice-design, then accept the invite.
⚠️
Passwords & 2FA: set these up yourself on GitHub — never share credentials or codes with anyone or any tool.
2
Setup · once

Clone the repo & identify yourself

Get the project onto your machine and tell git who you are.

# Identify yourself (office identity)
git config --global user.name "Your Name"
git config --global user.email "your.name@tiket.com"

# Download the project
git clone https://github.com/payprice-design/Push-and-Pull-Knowledge-Base.git
cd Push-and-Pull-Knowledge-Base
3
Loop · start of each task

Create a branch

Never work on main directly. Branch per task so work stays isolated and reviewable.

# Always start from an up-to-date main
git checkout main
git pull

# Create & switch to your task branch
git checkout -b feature/add-pricing-guide

Naming

  • feature/… — a new addition
  • fix/… — a correction
  • docs/… — documentation only
4
Loop · as you work

Commit your changes

A commit is a saved snapshot. Commit often, in small logical chunks.

git status              # see what changed
git add .               # stage all changes
git commit -m "Add pricing FAQ section"
💡
Good messages say what & why in the present tense: "Add pricing FAQ section", not "stuff".
5
Loop · share your work

Push, Pull & Sync

Move commits between your computer (local) and GitHub (remote).

💻 Localpush ☁️ GitHubpull 💻 Local

Push — send commits up

# First push of a branch links it with -u
git push -u origin feature/add-pricing-guide

# After that, just:
git push

Pull & Sync — stay current

# Get teammates' latest work on your branch
git pull

# Bring the newest main into your branch
git checkout main && git pull
git checkout feature/add-pricing-guide
git merge main
⚠️
Pull before you start each session to build on the newest version and avoid conflicts.
6
Loop · get it reviewed

Contribute & open a Pull Request

A PR asks teammates to review your branch and merge it into main.

Push branch Open PR Review Merge
  1. Push your branch, then open the repo on GitHub — click “Compare & pull request” (or Pull requests → New).
  2. Set base: main ← compare: your branch.
  3. Write a clear title and describe what changed and why; link issues with #123.
  4. Add reviewers on the right sidebar, then Create pull request.
  5. Address feedback by pushing more commits — the PR updates automatically.
  6. Once approved, Merge, then Delete branch to stay tidy.
7
Ship it

Deploy to Vercel

Vercel publishes the site automatically whenever main updates.

Merge → main Vercel builds 🌐 Live site

Connect the repo (once)

  1. Go to vercel.com/new and log in with your office GitHub account.
  2. Click Import → choose payprice-design/Push-and-Pull-Knowledge-Base.
  3. For a plain HTML site, keep defaults (no build command), then Deploy.

After that

  • Production: every merge to main updates the live URL.
  • Preview: every PR gets its own temporary URL — check it before merging.
# Optional — deploy from the terminal
npm i -g vercel
vercel              # preview
vercel --prod       # production
